### 1. MHSA: A Lightweight Framework for Mitigating Hallucinations via Steered Attention in LVLMs

- **作者**: Wei Ding, Yilin Li, Yudong Zhang, Ruobing Xie, Jiansheng Chen, Xingwu Sun, Yu Wang
- **机构**: Tsinghua University, Tencent, University of Science and Technology Beijing, University of Macau
- **链接**: [arXiv](https://arxiv.org/abs/2605.14966) | [PDF](https://arxiv.org/pdf/2605.14966)
- **来源**: 📄 arXiv 关键词检索

![](https://arxiv.org/html/2605.14966v1/x1.png)

- **核心方法**: MHSA 在 LVLM 的 cross-modal attention 层后插入一个轻量 3 层 MLP generator，训练时用 DHCP discriminator 的监督信号 + LVLM 自身的 token prediction 信号来学习"纠正后的 attention map"。推理时直接用纠正 attention 替换原始 cross-modal attention，不改 LVLM 任何参数。支持 discriminative（POPE）和 generative（captioning）两种幻觉场景。对 token-level 生成式幻觉做了扩展，可以逐 token 纠正。
- **对比方法/Baselines**: 基于 DHCP 的检测方法、[[OPERA]]、[[VCD]] 等 decoding 策略，在 [[POPE]]、[[CHAIR]] 上对比
- **借鉴意义**: 对做多模态幻觉抑制的人很有价值——不需要训练 LVLM 本身，也不需要外部 verifier，即插即用。缺点是只在 attention 层面操作，对那些"视觉信息足够但模型仍然胡说"的情况（语言 prior 过强）可能效果有限
- **锐评**: 思路干净：把 DHCP 从"检测"推到"纠正"，training 只训一个 3 层 MLP。实验覆盖了 dataset generalization、model generalization、cross-dataset OOD，表格里 F1 涨点看起来 solid。但说实话，3 层 MLP 能学到多少 attention correction 能力我是存疑的——cross-modal attention 在高维空间里的纠偏可能不是简单 MLP 能 capture 的。POPE 这种 binary 判断场景好办，真到长文本 captioning 里 token-level correction 会不会退化需要看 full paper 的 case study。另外 Tencent + Tsinghua 的组合，工程落地味道比较重。🔥
- 💡 **想精读？** 运行：`读一下 MHSA: A Lightweight Framework for Mitigating Hallucinations via Steered Attention in LVLMs`

### 2. Do We Really Need External Tools to Mitigate Hallucinations? SIRA: Shared-Prefix Internal Reconstruction of Attribution

- **作者**: Tian Qin et al.
- **机构**: Tsinghua University, Stanford University, The University of Sydney
- **链接**: [arXiv](https://arxiv.org/abs/2605.14621) | [PDF](https://arxiv.org/pdf/2605.14621)
- **来源**: 📄 arXiv 关键词检索

![](https://arxiv.org/html/2605.14621v1/x1.png)

- **核心方法**: SIRA 是 training-free 的 internal contrastive decoding 框架。关键 idea：利用 multimodal transformer 的分阶段信息流——先在 shared prefix 层让图文 token 充分交互形成对齐的 multimodal state；然后在后面某层 fork 出 counterfactual branch，mask 掉 image token 的 attention。这个 counterfactual branch 保留了 shared multimodal context 但失去了细粒度视觉信息，成为"语言 prior 主导"的内部 reference。解码时对比两个 branch 的 logits，suppress 掉不需要视觉也能预测好的 token。
- **对比方法/Baselines**: [[VCD]]、[[OPERA]]、[[DoLa]]、[[MaskCD]]、ICT、SID，在 [[POPE]]、[[CHAIR]]、[[AMBER]] 上评测
- **借鉴意义**: 比 VCD 聪明——不需要外部 perturbed input（VCD 要给图像加噪声），也不需要额外 forward pass（MaskCD 之类要跑两次）。直接在模型内部构造 counterfactual，overhead 低。对做 efficient inference-time hallucination mitigation 的人来说这是个好方向
- **锐评**: 标题就挑衅——"Do We Really Need External Tools?"——然后给了一个优雅的内部方案。单次 forward pass 内完成 contrastive decoding 确实比两轮推理的 VCD 省了一倍时间。实验在 Qwen2.5-VL 和 LLaVA-v1.5 上都做了，POPE/CHAIR/AMBER 三个 benchmark 全有。但我有几个疑问：boundary layer（在哪里 fork）需要 per-model tuning，不是真正的 plug-and-play；counterfactual branch 如果 mask 太早会丢掉 shared context，太晚又没区分度——这个 trade-off 是 SIRA 的阿喀琉斯之踵。另外 ablation 里不同 boundary 的敏感度有多少，需要看 Table 里的具体数字。🔥

### 3. When Answers Stray from Questions: Hallucination Detection via Question-Answer Orthogonal Decomposition

- **作者**: Yao Siyang, Feng Erhu, Xia Yubin
- **机构**: Shanghai Jiao Tong University
- **链接**: [arXiv](https://arxiv.org/abs/2605.14449) | [PDF](https://arxiv.org/pdf/2605.14449)
- **来源**: 📄 arXiv 关键词检索

![](https://arxiv.org/html/2605.14449v1/x1.png)

- **核心方法**: QAOD 把幻觉检测建模为"答案表示中与问题正交的分量"——先提取 answer hidden state，然后投影掉 question-aligned 方向，得到 question-orthogonal 分量。这个分量里的信号反映了答案是否偏离问题（即幻觉）。用 diversity-penalized Fisher scoring 选 layer、Fisher importance 选 neuron。设计了两个 probe：joint probe（orthogonal + question context，in-domain 最优）和 orthogonal-only probe（domain-agnostic，OOD 泛化最强）。
- **对比方法/Baselines**: [[SelfCheckGPT]]、[[PPL]]、MIND、SAPLMA 等 white-box/black-box 方法，在 TriviaQA、NQ、BioASQ 上评测
- **借鉴意义**: 对做多模态幻觉检测的人有一定参考——正交分解的思路可以迁移到 VLM 的 vision-language 对齐检测上。但这篇本身是纯 LLM 幻觉检测，不是多模态
- **锐评**: 方法动机很直觉——如果答案跟问题不相关，那答案表示中"问题无关的方向"应该很大。OOD 实验结果确实漂亮：orthogonal-only probe 在 BioASQ 上超最强 white-box baseline 21%，只用不到 25% 的生成成本。但这个框架对 hallucination 的定义隐含了"偏离问题 = 幻觉"，而有些幻觉是答案跟问题相关但事实错误——这种情况 QAOD 可能漏掉。另外纯 LLM 的实验结果能不能迁移到 LVLM 场景是个大问号。标题起得不错。👀

### 4. Where Does Reasoning Break? Step-Level Hallucination Detection via Hidden-State Transport Geometry

- **作者**: 待补充
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2605.15012) | [PDF](https://arxiv.org/pdf/2605.15012)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 用 hidden state 的 transport geometry 在推理链的每一步检测幻觉——不是判断整段输出是否幻觉，而是定位到具体哪一步推理开始出错。用最优传输（optimal transport）度量 hidden state 在相邻推理步之间的几何偏移，异常的 transport cost 对应幻觉步。
- **对比方法/Baselines**: 各类 hallucination detection probes
- **借鉴意义**: step-level 的粒度比 sentence-level 细得多，对做 reasoning-grounded VLM 的人有参考价值。如果能迁移到多模态 Chain-of-Thought 场景，可以用来诊断 VLM 在哪个视觉推理步上瞎编
- **锐评**: 步级幻觉检测是个 real problem——现在大部分方法只看最终输出有没有幻觉，但定位到具体推理步才能做 targeted intervention。Transport geometry 这个角度不算新（有人用 OT 分析过 representation dynamics），但用在幻觉检测上确实应景。担忧是：最优传输的计算成本不低，推理链一长 overhead 可能很大；另外"transport cost 大 = 幻觉"这个假设需要更多 justification——有时候模型在困难推理步本来就会有更大的 representation shift。摘要写得太少，很多细节需要看正文。👀

### 5. Deep Pre-Alignment for VLMs

- **作者**: Tianyu Yu, Kechen Fang, Zihao Wan, Kaidong Zhang, Yicheng Zhang, Jun Song, Bo Zheng
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2605.15305) | [PDF](https://arxiv.org/pdf/2605.15305)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 把 VLM 的 alignment 目标从最终输出层解耦出来，在中间层也引入对齐约束。核心思路是：不要只在最后一层做 vision-language alignment，而是在多个深度层次上都施加对齐监督，让视觉和语言表示在更早的阶段就建立对应关系。这样后续的 cross-modal reasoning 有更丰富的对齐信号可用。
- **对比方法/Baselines**: 各类 VLM pre-training alignment 策略
- **借鉴意义**: 对做 VLM 预训练的人有直接参考价值——如果 deep pre-alignment 确实有效，可以以很小的额外训练成本提升下游 hallucination resistance
- **锐评**: Deep alignment 的直觉是对的——只在最后一层做对齐就像只检查最终答案不检查中间推导。问题在于 deep supervision 在 CV 里已经被玩烂了（GoogLeNet 的 auxiliary classifier 都多少年了），这篇在 VLM 场景下的 novelty 在于把 deep supervision 用在 cross-modal alignment 上。但"deep alignment 减少幻觉"这个 claim 需要更多因果证据，不只是 correlation。实验部分摘要信息不够，不知道到底在哪些 benchmark 上验证了。👀

### 6. Unlocking Complex Visual Generation via Closed-Loop Verified Reasoning

- **作者**: Hanbo Cheng et al.
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2605.15273) | [PDF](https://arxiv.org/pdf/2605.15273)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 在 text-to-image 生成中引入 closed-loop verification——生成图像后，用一个 verifier 检查图像是否满足 prompt 中的所有语义约束，如果不满足则反馈给 generator 进行修正。本质是把 reasoning + self-correction 引入图像生成。
- **对比方法/Baselines**: 推理增强的 T2I 方法
- **借鉴意义**: closed-loop self-correction 的思路对做 embodied AI 里"规划→执行→验证"循环的人有启发。但用在 image generation 上 cost 会很高
- **锐评**: closed-loop verification 在 LLM agent 里很火，搬到 image generation 也算合理延伸。但 image generation 的 verification 比 text 难得多——text verifier 可以靠 NLI，image verifier 靠什么？CLIP score？那跟训练目标高度耦合，闭环可能只是强化了 CLIP 的 bias。另外每张图要 generate → verify → refine，推理成本至少翻倍。idea 方向对，但落地到实际产品可能要等 inference cost 再降一个数量级。👀

### 7. Reversing the Flow: Generation-to-Understanding Synergy in Large Multimodal Models

- **作者**: 待补充
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2605.14880) | [PDF](https://arxiv.org/pdf/2605.14880)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 探索多模态模型中"生成能力"和"理解能力"之间的正反馈——用图像生成任务来提升 VLM 的视觉理解能力。相当于反向利用生成能力来增强理解，打破了传统"先理解再生成"的单向范式。
- **对比方法/Baselines**: standard VLM understanding benchmarks
- **借鉴意义**: 对做 VLM 训练的人有参考——如果生成任务确实能提升理解（类似"教别人能加深自己理解"），那训练 pipeline 可以加一个 auxiliary generation objective
- **锐评**: generation-to-understanding 的方向有意思但不好做。核心挑战是：生成任务（如 text-to-image）和理解任务（如 VQA）的梯度信号可能互相干扰。如果能证明生成任务提供了某种"dense supervision"而不仅仅是额外的 noise，那才是有价值的 contribution。摘要写得比较模糊，具体是用什么 generation task、在哪些 understanding benchmark 上有提升，都不清楚。看完摘要感觉像是一个 training trick 包装成了 framework。⚠️

### 8. VLMs Trace Without Tracking: Diagnosing Failures in Visual Path Following

- **作者**: 待补充
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2605.14920) | [PDF](https://arxiv.org/pdf/2605.14920)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 研究 VLM 在视觉路径跟随任务中的失败模式。设计了简化版 Circuit Connection 任务和更复杂的路径跟随任务，分析 VLM 在需要"沿着视觉线索追踪"时的 failure case。发现 VLM 即使在简单场景下也会出现 tracing failure。
- **对比方法/Baselines**: 多个 VLM 模型的路径跟随能力对比
- **借鉴意义**: 对做 embodied AI 里 visual navigation 的人有用——VLM 如果连简单的 visual tracing 都做不好，那放到机器人上做视觉导航更悬
- **锐评**: 这个方向很实际——VLM 被吹得很神，但连"沿着线走"这种任务都可能翻车，说明 visual grounding 还有很大问题。论文贡献在于系统性地诊断了 failure mode，而不只是报告一个 aggregate score。但只做诊断不给解法，contribution 偏薄。诊断结果如果没有 actionable insight（比如"问题出在 patch embedding 的分辨率不够"），那只是又多了一篇"VLM 在 X 上不行"的 benchmarking paper。👀

### 9. LRCP: Low-Rank Compressibility Guided Visual Token Pruning for Efficient LVLMs

- **作者**: 待补充
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2605.14870) | [PDF](https://arxiv.org/pdf/2605.14870)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 用 low-rank compressibility 作为准则来 prune visual token——不是 uniform prune，而是评估每个 visual token 的 low-rank 可压缩性，保留信息量大的 token、丢弃可压缩的冗余 token。在不损失视觉信息的前提下减少 LLM 处理的 token 数量。
- **对比方法/Baselines**: FastV、ToMe 等 token pruning 方法
- **借鉴意义**: 对需要在边缘设备上跑 VLM 的人有用。但如果你的场景是服务器推理，token pruning 带来的加速可能不如 quantization 或 KV cache optimization 显著
- **锐评**: Visual token pruning 这个赛道已经很拥挤了——FastV、ToMe、PruMerge、VisionZip……LRCP 的 novelty 在于用 low-rank 作为 pruning criterion 而不是 attention score。但这本质上是用一个 heuristic 替换另一个 heuristic。真正的问题是：prune 掉的 token 在需要细粒度 visual grounding 时会不会是关键信息？所有 token pruning 方法的命门都在这里——在 VQA benchmark 上不掉点不代表在需要精确定位的任务上不掉点。👀

### 12. UAM: A Dual-Stream Perspective on Forgetting in VLA Training

- **作者**: 待补充
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2605.14842) | [PDF](https://arxiv.org/pdf/2605.14842)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 从 dual-stream 视角分析 VLA 训练中的灾难性遗忘——把 VLA 的表示空间分解为 vision-language stream 和 action stream，分析在 fine-tuning 过程中两个 stream 各自的遗忘模式。发现不同 stream 的遗忘速率和模式不同，需要不同的 mitigation 策略。
- **对比方法/Baselines**: 标准的 continual learning / anti-forgetting 方法在 VLA 场景下的对比
- **借鉴意义**: 对做 VLA / robot foundation model 训练的人非常重要——如果你在通用 VLM 上 fine-tune robot policy，一定会遇到"学会了 action 但忘了 visual understanding"或者反过来。UAM 提供的 dual-stream 分析框架可以帮助设计更好的 fine-tuning 策略
- **锐评**: VLA 的灾难性遗忘是个 real & painful 的问题——想象一下你在 VLM 上 fine-tune manipulation policy，结果模型学会了抓杯子但忘了"杯子"长什么样（visual grounding 退化了）。UAM 的 dual-stream 分解是合理的第一步分析框架。但论文贡献可能止步于"发现问题并分析"而不给出有效的 mitigation 方法（从标题看是 analysis paper）。如果 ablation 里有尝试不同的 anti-forgetting 策略并给出 recommendation，那价值会大很多。🔥

### 13. WorldAct: Activating Monolithic 3D Worlds into Interactive-Ready Object-Centric Scenes

- **作者**: 待补充
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2605.15137) | [PDF](https://arxiv.org/pdf/2605.15137)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 从 monolithic 3D 场景（如 NeRF/3DGS 重建的完整场景）中提取 object-centric 的交互就绪表示。把一个完整的 3D 世界分解成可以独立操作的物体，使 agent 可以在其中进行物体级的交互（抓取、推动、打开等）。
- **对比方法/Baselines**: 3D scene decomposition、object-centric learning 方法
- **借鉴意义**: 对做 embodied AI simulation 和 world model 的人很有价值——如果能把 3D 场景自动分解成交互就绪的物体，那 robot learning 的仿真环境构建效率会大幅提升
- **锐评**: "monolithic 3D world → interactive object-centric scenes"的定位很清晰。但 object-centric decomposition 在 2D 图像上已经研究了很久（slot attention 那一套），搬到 3D 场景里主要看"交互就绪"的定义是否合理。如果分解出的 object 只是几何意义上的分割，但没有 physical property（质量、摩擦、可抓取性），那离"交互就绪"还有距离。标题里的"Activating"用得好——让人想看看到底怎么 activate。👀

- **机构**: 未知

### 16. Unlocking Dense Metric Depth Estimation in VLMs

- **作者**: 待补充
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2605.14927) | [PDF](https://arxiv.org/pdf/2605.14927)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 想办法从 VLM 中提取 dense metric depth——不是"这个物体比那个远"的相对深度，而是有实际尺度的 metric depth。利用 VLM 内部的多尺度视觉特征来重建深度图。
- **对比方法/Baselines**: 专门的深度估计模型（如 DepthAnything、ZoeDepth）
- **借鉴意义**: 对做 embodied AI 3D perception 的人有参考——如果 VLM 本身就能输出 metric depth，那就不需要单独的深度传感器或深度估计模型，减少 pipeline 复杂度
- **锐评**: VLM 做 depth estimation 的思路有趣但挑战很大。VLM 的 visual encoder 通常被 CLIP-style contrastive loss 训练过，保留的是语义信息而不是几何信息。从语义特征里挖 metric depth 本质上是"从经过信息瓶颈压缩的表示中恢复被丢弃的几何信号"。能 work 的话说明 CLIP 特征里残留的几何信息比想象的多；不能 work 也正常。跟专门做 depth 的模型（DepthAnything V2 等）比大概率会输，但"不输太惨"本身就值得关注。👀
