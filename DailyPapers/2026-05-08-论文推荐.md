### 1. Beyond SFT-to-RL: Pre-alignment via Black-Box On-Policy Distillation for Multimodal RL
- **作者**: Sudong Wang, Weiquan Huang, Xiaomin Yu, Zuhao Yang, Hehai Lin, Keming Wu, Chaojun Xiao, Chen Chen, Wenxuan Wang, Beier Zhu, Yunjian Zhang, Chengwei Qin
- **机构**: HKUST (Guangzhou), Tsinghua University, NTU, RUC, USTC, UCAS
- **链接**: [arXiv](https://arxiv.org/abs/2604.28123) | [PDF](https://arxiv.org/pdf/2604.28123)
- **来源**: 🔥 HF Trending，⬆️ 41

![](https://arxiv.org/html/2604.28123v2/x1.png)

- **核心方法**: PRISM 提出三阶段流水线：SFT → Distribution Alignment → RLVR。核心是在 SFT 和 RL 之间插入一个显式的分布对齐阶段，用 black-box on-policy distillation（OPD）来纠正 SFT 引入的 distributional drift。具体做法是训练一个 MoE discriminator（perception expert + reasoning expert），让 policy 和 discriminator 做对抗游戏，不需要 teacher logits。用了 1.26M 公开数据做 SFT 初始化，又从 Gemini 3 Flash 精选了 113K 高质量数据做分布对齐。
- **对比方法/Baselines**: 对比了 [[GRPO]]、[[DAPO]]、[[GSPO]] 等 RL 方法；借鉴了 on-policy distillation 的思想但做了 black-box 改造
- **借鉴意义**: 对做多模态推理、VLM post-training 的人很有价值。SFT → RL 这个范式大家都在用，但 SFT 的 distributional drift 问题一直被忽视，这篇给出了一个具体的解法。MoE discriminator 的设计（perception/reasoning 解耦）也值得参考。
- **锐评**: 方法有新意，实验也扎实（MMMU、MathVerse、MathVision、WeMath 四个 benchmark 都有提升）。但有两个问题：1）依赖 Gemini 3 Flash 精选的 113K 数据，这数据不开源的话别人很难复现；2）black-box discriminator 的效果可能不如 white-box，作者没有做这个对比。另外 MoE discriminator 的训练成本没提，这块可能不便宜。🔥
- **关联笔记**: 无

### 2. Tuna-2: Pixel Embeddings Beat Vision Encoders for Multimodal Understanding and Generation
- **作者**: Zhiheng Liu, Weiming Ren, Xiaoke Huang, Shoufa Chen, Tianhong Li, Mengzhao Chen, Yatai Ji, Sen He, Jonas Schult, Belinda Zeng, Tao Xiang, Wenhu Chen, Ping Luo, Luke Zettlemoyer, Yuren Cong
- **机构**: Meta AI, The University of Hong Kong, University of Waterloo
- **链接**: [arXiv](https://arxiv.org/abs/2604.24763) | [PDF](https://arxiv.org/pdf/2604.24763)
- **来源**: 🔥 HF Trending，⬆️ 68

![](https://arxiv.org/html/2604.24763v1/x1.png)

- **核心方法**: Tuna-2 是一个完全 encoder-free 的统一多模态模型，理解和生成都在 pixel space 做。核心改动：1）用简单的 patch embedding 替代预训练的 vision encoder（如 SigLIP）；2）用 masked feature learning 来学习视觉表征；3）理解和生成共享同一个 visual tokenizer，避免表征不对齐。训练分三阶段：understanding pre-training → generation pre-training → joint SFT。
- **对比方法/Baselines**: 对比了 [[BAGEL]]、[[OmniGen2]]、[[UniVision]]、[[UniWorld]] 等统一模型；也对比了 [[SigLIP]] encoder-based 方案
- **借鉴意义**: 对做统一多模态模型的人是必读。Encoder-free 这个方向之前有人探索过（如 Early LLMs），但 Tuna-2 是第一个在理解和生成两个任务上都达到 SOTA 的。Masked feature learning 的设计也值得借鉴。
- **锐评**: 实验很全面（GenEval、MMVP、ImgEdit 都有），而且消融实验证明了 pixel embedding 确实比 SigLIP encoder 好——这个结论有点反直觉，作者解释是 encoder 的预训练目标和下游任务不匹配，而 pixel embedding 可以端到端优化。但有个问题：训练成本。作者说用了 1.2M GPU hours，这个门槛太高了，普通人玩不起。另外 encoder-free 是否真的比 encoder-based 更好，还需要更多工作验证，一篇论文不够。🔥
- **关联笔记**: 无

### 3. UniVidX: A Unified Multimodal Framework for Versatile Video Generation via Diffusion Priors
- **作者**: Houyuan Chen, Hong Li, Xianghao Kong, Tianrui Zhu, Shaocong Xu, Weiqing Xiao, Yuwei Guo, Chongjie Ye, Lvmin Zhang, Hao Zhao, Anyi Rao
- **机构**: Nanjing University, MMLab@CUHK, MMLab@HKUST, Tsinghua University, BAAI, Stanford University
- **链接**: [arXiv](https://arxiv.org/abs/2605.00658) | [PDF](https://arxiv.org/pdf/2605.00658)
- **来源**: 🔥 HF Trending，⬆️ 80

![](https://arxiv.org/html/2605.00658v1/x1.png)

- **核心方法**: UniVidX 用一个统一的 diffusion 框架处理多种视频生成任务（text-to-video、image-to-video、video editing、intrinsic decomposition 等）。核心设计：1）Stochastic Condition Masking（SCM）来统一不同任务的输入格式；2）Decoupled Gated LoRA（DGL）来处理不同任务的条件注入，避免任务间干扰。底层用的是预训练的 VDM prior。
- **对比方法/Baselines**: 对比了 [[FLUX]]、[[LayerDiffuse]]、[[IntrinsiX]]、[[NormalCrafter]] 等任务特定模型；借鉴了 LoRA 的思想但做了 decoupled 改造
- **借鉴意义**: 对做视频生成、多任务统一框架的人有参考价值。SCM 和 DGL 的设计比较巧妙，可以迁移到其他多任务场景。
- **锐评**: 80 个 upvotes 说明社区认可度不错。方法层面 SCM + DGL 的组合确实解决了多任务统一的一个核心问题（任务间干扰），实验也覆盖了 6 个任务。但有个问题：每个任务都需要单独训练 LoRA，虽然参数量小但管理成本高。另外论文没有和真正的 multi-task model（如一个模型同时做所有任务）做对比，只和 task-specific model 比，这个 baseline 选择有点讨巧。👀

### 4. D-OPSD: On-Policy Self-Distillation for Continuously Tuning Step-Distilled Diffusion Models
- **作者**: Dengyang Jiang, Z-Image Team
- **机构**: HKUST, Alibaba Group, UCSD, CUHK
- **链接**: [arXiv](https://arxiv.org/abs/2605.05204) | [PDF](https://arxiv.org/pdf/2605.05204)
- **来源**: 🔥 HF Trending，⬆️ 21

![](https://arxiv.org/html/2605.05204v1/x1.png)

- **核心方法**: D-OPSD 解决的是 few-step diffusion model（如 FLUX.1、Z-Image-Turbo）的持续微调问题。核心思想：用 on-policy self-distillation 来保持模型的 few-step 推理能力。具体做法：1）用当前模型生成样本；2）用这些样本做 self-distillation；3）用 PSO（Particle Swarm Optimization）来优化蒸馏目标。支持 LoRA 和 full finetuning 两种模式。
- **对比方法/Baselines**: 对比了 [[SFT]]、[[DPO]]、[[DreamBooth]]、[[ReFL]] 等微调方法；借鉴了 on-policy distillation 的思想
- **借鉴意义**: 对做 diffusion model 微调、个性化生成的人有实用价值。Few-step model 的微调确实是个痛点，这篇给出了一个可行的方案。
- **锐评**: 方法实用，实验也比较充分（在 FLUX.1 和 Z-Image-Turbo 上都验证了）。但有两个问题：1）PSO 的引入有点突兀，作者没有解释为什么用 PSO 而不是其他优化方法；2）on-policy self-distillation 的计算成本不低（需要用当前模型生成样本），这个 trade-off 没有量化分析。另外 21 个 upvotes 不算多，社区认可度一般。👀
- **关联笔记**: 无

### 5. Hallucination as an Anomaly: Dynamic Intervention via Probabilistic Circuits
- **作者**: Nielsen, Erik, Cunegatti, Elia, Vukojevic, Marcus, Iacca, Giovanni
- **机构**: University of Trento
- **链接**: [arXiv](http://arxiv.org/abs/2605.05953v1) | [PDF](http://arxiv.org/pdf/2605.05953v1)
- **来源**: 📄 arXiv 关键词检索

![](https://arxiv.org/html/2605.05953v1/x1.png)

- **核心方法**: PCNET 把幻觉检测建模为异常检测问题。核心设计：1）用 Probabilistic Circuit（PC）在 LLM 的 residual space 上训练一个 tractable density estimator；2）通过 PC 的 exact marginal inference 来检测 anomalous tokens；3）只对检测到的 anomalous tokens 做干预，避免误伤正确生成。PC 的好处是可以做 exact inference，不需要采样。
- **对比方法/Baselines**: 对比了 [[DoLa]]、[[AdaSteer]]、[[ICD]]、[[ITI]] 等幻觉检测方法；借鉴了 probabilistic modeling 的思想
- **借鉴意义**: 对做幻觉检测、LLM 可靠性的人有参考价值。把幻觉建模为异常检测这个视角比较新颖，PC 的 exact inference 也是一个技术亮点。
- **锐评**: 方法有新意，把 PC 引入幻觉检测是个好想法。但有几个问题：1）PC 的表达能力有限，只能建模简单的分布，对于 LLM residual space 这种高维复杂分布可能不够；2）PC 的训练成本没提，这块可能不便宜；3）只在 TriviaQA、TruthfulQA、CoQA 上验证，benchmark 覆盖面不够。另外作者来自 Trento，不是主流 LLM 研究组，需要看代码才能判断工程质量。👀
- **关联笔记**: 无
- 💡 **想精读？** 运行：`读一下 PCNET`
