## 多模态幻觉：检测与抑制

### 1. Mitigating Multimodal LLMs Hallucinations via Relevance Propagation at Inference Time
- **作者**: Itai Allouche, Joseph Keshet
- **机构**: Technion – Israel Institute of Technology（从作者邮箱推断）
- **链接**: [arXiv](https://arxiv.org/abs/2605.01766) | [PDF](https://arxiv.org/pdf/2605.01766)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[LIME]]

- **核心方法**: LIME（Learning Inference-time Modality Enhancement）是一个 training-free 框架，核心思路是用 Layer-wise Relevance Propagation（LRP）量化每个 token 对输出的贡献度，然后定义一个 relevance-based objective 在解码时提升感知 token（图像/音频）的权重，抑制文本先验的支配地位。不需要额外训练、不需要外挂 verifier，纯推理时干预。
- **对比方法/Baselines**: 对比了 [[DoLa]]、[[ICD]] 等 decoding-based 幻觉抑制方法，以及与 [[ITI]]、[[SADI]] 等 representation engineering 路线的差异。
- **借鉴意义**: 对做 VLM 部署的人非常实用——training-free、推理时开销可控。LRP 的 token 贡献度分析思路也可以迁移到其他 VLM 可解释性问题。
- **锐评**: 思路干净——MLLM 幻觉的根因之一是文本 token 在 attention 中压倒视觉 token，用 LRP 量化这个 bias 然后反向纠偏，逻辑通顺。但论文目前只在有限 benchmark 上验证，且 LRP 的 overhead 在实际部署中是否可接受需要看具体数字。如果 ablation 做得到位、跨模型泛化性好，这是篇有实际价值的工作。🔥
- 💡 **想精读？** 运行：`读一下 论文 Mitigating Multimodal LLMs Hallucinations via Relevance Propagation at Inference Time`

### 2. CAST: Mitigating Object Hallucination in Large Vision-Language Models via Caption-Guided Visual Attention Steering
- **作者**: Qiming Li, Zekai Ye, Xiaocheng Feng, Weihong Zhong, Libo Qin, Ruihan Chen, Lei Huang, Baohang Li, Kui Jiang, Yaowei Wang, Ting Liu, Bing Qin
- **机构**: 哈尔滨工业大学（HIT）
- **链接**: [arXiv](https://arxiv.org/abs/2605.04641) | [PDF](https://arxiv.org/pdf/2605.04641)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: CAST 观察到一个关键现象：LVLM 在回答 caption 类 query 时对视觉信息的 attention 远强于非 caption query。基于此，CAST 用 probing 识别出对 caption query 高度敏感的 attention head，在非 caption query 时主动将这些 head 的输出向 caption 模式的方向 steering。Training-free，plug-and-play，不改模型权重。
- **对比方法/Baselines**: 对比了 [[VCD]]、[[OPERA]] 等 decoding 策略方法，以及需要训练的偏好对齐方法。和 [[GEASS]] 同为 caption-guided 路线但机制不同。
- **借鉴意义**: attention head 的 caption-sensitivity probing 方法本身很有价值，可以用于诊断其他 VLM 行为。steering 方向只需要少量 caption 样本标定，实用性好。
- **锐评**: "caption query 时模型看得更仔细"这个观察不新鲜，但把它系统化成 attention steering 的方向向量是聪明的 engineering。问题在于：caption head 的 steering 是否对所有类型的 query 都有效？对于需要空间推理、计数的 query，caption-style attention 可能反而不够——模型需要看的不是"有什么物体"而是"物体之间的关系"。需要看论文里是否区分了不同 query type。👀
- 💡 **想精读？** 运行：`读一下 论文 CAST: Mitigating Object Hallucination in Large Vision-Language Models via Caption-Guided Visual Attention Steering`

### 3. GEASS: Training-Free Caption Steering for Hallucination Mitigation in Vision-Language Models
- **作者**: Zeshang Li, Shuoyang Zhang, Jiashen Ding
- **机构**: 未知（摘要未列机构）
- **链接**: [arXiv](https://arxiv.org/abs/2605.01733) | [PDF](https://arxiv.org/pdf/2605.01733)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: GEASS（Gated Evidence-Aware Selective Steering）与其他 caption-guided 方法的关键区别是：它不假设 self-generated caption 总是有益的。论文发现 naive 地嵌入 caption 反而会让 Qwen2.5-VL-3B 在 HallusionBench 上掉近 10 个点。原因是 caption 中的 omission（遗漏物体）远多于 fabrication（编造物体），但每次 fabrication 的伤害更大。GEASS 用 gate 机制：clean path 的置信度决定 caption 用多少，caption 的熵减量决定其权重，两条路径分歧大时提高证据门槛。
- **对比方法/Baselines**: 对比了直接将 caption 注入 prompt 的基线、以及 [[VCD]] 等 decoding 方法。
- **借鉴意义**: "caption 不是银弹"这个发现本身就有价值——很多人在 pipeline 里无脑加 caption，GEASS 说明了为什么这可能会更差，以及怎么选择性使用。
- **锐评**: 这篇比 CAST 更有深度。发现了 caption errors 的不对称性（omission vs fabrication 的伤害不对称），并设计了对应的自适应门控。不是简单的"加 caption 就好了"，而是"什么时候加、加多少、加错了怎么办"。方法层面也更优雅。唯一担心的是 gate 的超参数对跨模型迁移是否鲁棒。👀
- 💡 **想精读？** 运行：`读一下 论文 GEASS: Training-Free Caption Steering for Hallucination Mitigation in Vision-Language Models`

### 4. Online Self-Calibration Against Hallucination in Vision-Language Models
- **作者**: Minghui Chen, Chenxu Yang, Hengjie Zhu, Dayan Wu, Zheng Lin, Qingyi Si
- **机构**: 未知（摘要未列完整机构）
- **链接**: [arXiv](https://arxiv.org/abs/2605.00323) | [PDF](https://arxiv.org/pdf/2605.00323)
- **来源**: 📰 HF Daily，⬆️ 3

- **核心方法**: OSCAR 的核心发现是 LVLM 中存在 Generative-Discriminative Gap——模型做判别（"图中是否有 X？"）的准确率高于做生成（"描述这张图"）。利用这个 gap，OSCAR 用 Monte Carlo Tree Search + Dual-Granularity Reward Mechanism 自动构建偏好数据，然后用 DPO 迭代优化模型。整个过程是 online self-supervised 的，不需要 GPT 等强模型蒸馏。
- **对比方法/Baselines**: 对比了需要 GPT 蒸馏的 offline preference alignment 方法（如 [[RLHF-V]]、[[POVID]] 等）。
- **借鉴意义**: Generative-Discriminative Gap 是一个可以在自己模型上快速验证的假设——先测你的 VLM 在判别和生成任务上的准确率差异，如果 gap 大，OSCAR 路线就有价值。self-calibration 思路比蒸馏 GPT 更可持续。
- **锐评**: 思路很正——用模型自己较强的判别能力去校准较弱的生成能力，避免了"用 GPT 教小模型去猜"的问题。MCTS + 双粒度 reward 的设计看起来合理但工程复杂度不低。需要确认的是：MCTS 的推理开销在在线 self-calibration 场景下是否可接受？如果每次校准都要跑几十次 rollout，性价比存疑。另外 Gap 的大小可能随模型和任务变化，不一定总是存在。👀
- 💡 **想精读？** 运行：`读一下 论文 Online Self-Calibration Against Hallucination in Vision-Language Models`

### 5. When Relations Break: Analyzing Relation Hallucination in Vision-Language Model Under Rotation and Noise
- **作者**: Philip Wootaek Shin, Ajay Narayanan Sridhar, Sivani Devarapalli, Rui Zhang, Jack Sampson, Vijaykrishnan Narayanan
- **机构**: Penn State University
- **链接**: [arXiv](https://arxiv.org/abs/2605.05045) | [PDF](https://arxiv.org/pdf/2605.05045)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 系统研究了视觉扰动（旋转和噪声）对 VLM 关系幻觉的影响。发现即使轻微的旋转/噪声也会显著降低跨模型、跨数据集的关系推理能力。对 prompt augmentation、orientation correction、denoising 等防御策略做了评估，结论是这些都只能部分缓解，无法根本解决问题。
- **对比方法/Baselines**: 在多个 VLM 和 VQA 数据集上评估。
- **借鉴意义**: 这是一篇分析性论文（analytical），不是方法论文。对做 VLM 鲁棒性的人有参考价值——它揭示了 perceptual robustness 和 relational understanding 之间的 gap：即使模型能"看清"物体，也未必能"理解"它们之间的关系。这个 gap 是后续方法设计的靶点。
- **锐评**: Analysis 做得扎实但贡献偏薄。发现"旋转和噪声导致关系幻觉"不够 surprising——这说明 VLM 的关系推理不是 geometry-aware 的，有点像 blind reasoning based on semantic priors。论文如果能进一步分析 WHY 旋转破坏了关系推理（是位置编码的问题？是训练数据缺乏 rotated samples？是 attention 的 spatial bias？），价值会大得多。现在停在现象描述层面。"prompt augmentation 只能部分缓解"这个结论对社区有用。⚠️

### 6. Hallucinations Undermine Trust; Metacognition is a Way Forward
- **作者**: Gal Yona, Mor Geva, Yossi Matias
- **机构**: Google
- **链接**: [arXiv](https://arxiv.org/abs/2605.01428) | [PDF](https://arxiv.org/pdf/2605.01428)
- **来源**: 📰 HF Daily，⬆️ 18

- **核心方法**: 一篇观点/立场论文（position paper）。核心论点：当前的事实性提升主要来自扩大模型的 knowledge boundary（编码更多事实），而非改善对边界的 awareness（区分已知与未知）。后者的根本困难在于模型缺少足够的判别能力来完美区分真伪，因此消除幻觉和保持有用性之间存在 tradeoff。论文提出改换框架：将幻觉定义为 confident errors，目标是让模型输出带上合适的置信度（metacognition），而不是消灭所有错误。
- **对比方法/Baselines**: N/A（观点论文）
- **借鉴意义**: metacognition 的 framing 对设计幻觉评估指标有启发——不应只看事实正确率，还应看模型的 confidence calibration 能力。如果做 VLM 的 hallucination benchmark，这个视角可以指导 metric 设计。
- **锐评**: Google 作者、观点不俗，但是一篇纯观点文——没有实验、没有方法、没有系统验证。论点本身不新鲜（uncertainty estimation 和 calibration 是 ML 老话题了），套到 LLM 幻觉上换个说法。它提出的 framing 确实有价值——"confident errors"的区分比笼统的"hallucination"更细粒度——但需要后续工作来 validate。当成思想框架看可以，不要期待 actionable 的方法。💤

### 7. Erase Persona, Forget Lore: Benchmarking Multimodal Copyright Unlearning in Large Vision Language Models
- **作者**: JuneHyoung Kwon, JungMin Yun, YoungBin Kim
- **机构**: KAIST（从作者姓名推断）
- **链接**: [arXiv](https://arxiv.org/abs/2605.03547) | [PDF](https://arxiv.org/pdf/2605.03547)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: CoVUBench，第一个专门评估 LVLM 版权内容遗忘的 benchmark。使用 procedurally generated 的合成数据（法律安全！）配合系统性的视觉变化（compositional changes + diverse domain manifestations）来评估 unlearning 的泛化能力。评估协议覆盖遗忘效果和保留能力两个维度。
- **对比方法/Baselines**: 对比了多种 unlearning 方法在 LVLM 上的表现。
- **借鉴意义**: 合成数据 + 系统性视觉变化的 benchmark 设计思路值得参考——既避免了直接使用版权内容的合规问题，又能制造足够多样的测试场景。对于做 multimodal unlearning 的人是当前最好的评估工具。
- **锐评**: Benchmark 论文的通病——造了个不错的靶子但自己不下场。合成数据和真实版权内容的 gap 是多少？用 procedurally generated characters 测出来的泛化性能不能迁移到真实世界的 Mickey Mouse 和 Pokémon？这是 CoVUBench 的根本有效性问题。不过话说回来，直接用版权内容做 benchmark 本身有法律风险，合成数据可能是目前唯一可行的路线。Benchmark 设计精细度看起来不错。👀
- 💡 **想精读？** 运行：`读一下 论文 Erase Persona, Forget Lore: Benchmarking Multimodal Copyright Unlearning in Large Vision Language Models`

### 8. Uncovering Entity Identity Confusion in Multimodal Knowledge Editing
- **作者**: Shu Wu, Xiaotian Ye, Xinyu Mou, Dongsheng Liu, Xiaohan Wang, Mengqi Zhang
- **机构**: 中科院自动化所（CASIA）
- **链接**: [arXiv](https://arxiv.org/abs/2605.06096) | [PDF](https://arxiv.org/pdf/2605.06096)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 发现多模态知识编辑（MKE）后的模型会出现 Entity Identity Confusion（EIC）——纯文本问"原实体是谁"，模型会返回新实体的信息。构建了 EC-Bench 诊断 benchmark，分析发现根因在于现有 MKE 方法无法区分 Image-Entity binding 和 Entity-Entity relational knowledge，模型走捷径学到的是 E-E association 而非正确的 I-E binding。还探索了可能的缓解方向。
- **对比方法/Baselines**: 对比了 [[FT]]、[[MEND]]、[[SERAC]]、[[IKE]] 等知识编辑方法。
- **借鉴意义**: EIC 是一个重要的 failure mode 发现——做 MKE 的人之前应该都没想到编辑后的模型会有这种"身份混淆"。EC-Bench 的诊断方法可以用于评估任何 MKE 方法的编辑质量。区分 I-E binding 和 E-E relational knowledge 的分析框架有理论价值。
- **锐评**: 问题发现比方法贡献大。EIC 这个 failure mode 找得准，诊断分析做得细致。但论文在"怎么解决"上不给力——提出了缓解方向但没有给出真正有效的解决方案。这其实挺能说明问题的：当前的 MKE 方法（FT、MEND、SERAC 等）从根本上就没有建模 I-E vs E-E 的区分，打补丁可能不够，需要重新设计编辑机制。是一篇好 analysis paper，留给后续方法论文巨大的改进空间。👀
- 💡 **想精读？** 运行：`读一下 论文 Uncovering Entity Identity Confusion in Multimodal Knowledge Editing`

### 9. ICU-Bench: Benchmarking Continual Unlearning in Multimodal Large Language Models
- **作者**: Yuhang Wang, Wenjie Mei, Junkai Zhang, Guangyu He, Zhenxing Niu, Haichang Gao
- **机构**: 西安电子科技大学（Xidian University）
- **链接**: [arXiv](https://arxiv.org/abs/2605.05938) | [PDF](https://arxiv.org/pdf/2605.05938)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: ICU-Bench 聚焦 continual unlearning——不是一次性忘记一件事，而是持续收到遗忘请求（像 GDPR 的持续合规要求）。包含 1000 个隐私敏感 profiles（医疗报告+劳动合同），9500 张图片，16000 个 QA pairs，100 个 forget tasks。新的 metrics 覆盖遗忘效果、历史遗忘保持、保留有用性、持续遗忘过程中的稳定性。
- **对比方法/Baselines**: 评估了 [[GA]]、[[NPO]] 等 unlearning 方法在 continual setting 下的表现。
- **借鉴意义**: continual unlearning 的设定比 one-shot unlearning 更贴近实际部署场景（尤其是 GDPR 合规），benchmark 和 metrics 对做 MLLM 安全的人有直接用途。
- **锐评**: Benchmark 设计合理，continual 设定确实更真实——现实中遗忘请求是一个接一个来的，不是一次性全忘。但 medical reports + labor contracts 的数据选择太窄了，是不是因为这两个领域的数据相对好获取？还是因为隐私 concerns 只在这两个领域突出？benchmark 的领域泛化性存疑。另外，4 位作者 1 个机构 benchmark 的规模是否足够大？⚠️
- 💡 **想精读？** 运行：`读一下 论文 ICU-Bench:Benchmarking Continual Unlearning in Multimodal Large Language Models`

### 10. Metric Unreliability in Multimodal Machine Unlearning: A Systematic Analysis and Principled Unified Score
- **作者**: Abdullah Ahmad Khan, Hamid Laga, Ferdous Sohel
- **机构**: Murdoch University, Australia
- **链接**: [arXiv](https://arxiv.org/abs/2605.02206) | [PDF](https://arxiv.org/pdf/2605.02206)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 第一篇系统研究 VLM unlearning 指标可靠性的论文。在 3 个 VQA benchmark 上用 36 个 unlearned LLaVA-1.5-7B 模型，发现 5 个标准指标（FA、RA、MIA、AD、JS）分成两个互斥的簇：{FA, RA, MIA} 和 {AD, JS}，Kendall tau = -0.26。多模态 VQA 中指标分歧比单模态分类更严重。提出了 Unified Quality Score（UQS）作为综合指标。
- **对比方法/Baselines**: 在 LLaVA-1.5-7B 和 BLIP-2 OPT-2.7B 上验证。
- **借鉴意义**: 如果你在做 VLM unlearning，这篇必须看——你用的指标可能排名结果和别的指标完全相反。UQS 提供了一个更统一的评估方案。多模态场景下指标分歧更大的发现说明 cross-modal interaction 确实让 unlearning 评估更难。
- **锐评**: 扎实的元分析工作。36 个模型的实验量不小，两个簇的发现清晰有力。这不是那种 exciting 的方法论文，但它解决了一个真实问题——每个人用不同指标评估 unlearning，结果彼此矛盾，这个领域需要一个共识。UQS 的提出是务实的 engineering solution。⚠️ 扣分是因为贡献本质上是一个 empirical finding + weighted average，没有理论证明为什么 UQS 的权重选择是最优的。
- 💡 **想精读？** 运行：`读一下 论文 Metric Unreliability in Multimodal Machine Unlearning`

## 多模态内部机制与可解释性

### 11. Causal Probing for Internal Visual Representations in Multimodal Large Language Models
- **作者**: Zehao Deng, Tianjie Ju, Zheng Wu, Liangbo He, Jun Lan, Huijia Zhu, Weiqiang Wang, Zhuosheng Zhang
- **机构**: 上海交通大学（从作者背景推断）
- **链接**: [arXiv](https://arxiv.org/abs/2605.05593) | [PDF](https://arxiv.org/pdf/2605.05593)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 用 causal activation steering 系统性地探测 MLLM 内部的视觉概念编码。通过干预四类视觉概念（实体、抽象概念等），发现实体（entities）有局部化的专属记忆区域，而抽象概念是全局分布式的。关键发现：模型深度对抽象概念编码至关重要（scaling law 的 mechanistic driver），但实体定位几乎不受 scale 影响。反向 steering 实验揭示了感知与输出之间的补偿机制——抑制输出会触发隐层激活激增。
- **对比方法/Baselines**: N/A（分析性工作）
- **借鉴意义**: 对理解 MLLM 如何编码视觉信息有直接价值——尤其是实体 vs 抽象概念的不同编码机制。如果你在做 VLM 的可解释性或想设计更好的视觉 token 压缩方案，这个 insight（实体局部化、抽象全局化）可以帮助你决定哪些层更重要。
- **锐评**: 这是一篇正经的 mechanistic interpretability 工作，不是那种"我们看了一下 attention map"的浅层分析。causal steering 比 correlational probing 更有说服力。"实体局部化、抽象全局化"的发现干净利落，reverse steering 的补偿机制发现也很有意思。但与 hallucination 的关联不够直接——它能帮我们理解"模型是怎么看图的"，但距离"模型为什么胡说"还有一段距离。把这个 framework 扩展到 hallucination detection 会很有前景。👀
- 💡 **想精读？** 运行：`读一下 论文 Causal Probing for Internal Visual Representations in Multimodal Large Language Models`

## LLM 幻觉：方法与分析（通用）

### 12. Hallucination as an Anomaly: Dynamic Intervention via Probabilistic Circuits
- **作者**: Erik Nielsen, Elia Cunegatti, Marcus Vukojevic, Giovanni Iacca
- **机构**: University of Trento
- **链接**: [arXiv](https://arxiv.org/abs/2605.05953) | [PDF](https://arxiv.org/pdf/2605.05953)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: PCNET——在 LLM 残差流上训练一个 Probabilistic Circuit 作为可处理密度估计器（tractable density estimator）。用精确 NLL 计算检测幻觉（把幻觉视为 factual manifold 上的几何异常），无需采样、外部验证器或修改权重。PC-LDCD 只在检测到偏离时才触发纠正，避免破坏正确生成的 token。在 4 个 LLM（1B-8B）、4 个 benchmark 上，AUROC 最高达 99%，TruthfulQA 上 True+Info 最高，mean corruption rate 降到 53.7%。
- **对比方法/Baselines**: 对比了 [[DoLa]]、[[ICD]]、[[ITI]]、[[SADI]] 等 decoding/representation 方法。
- **借鉴意义**: Probabilistic Circuit 做密度估计 + 异常检测的思路干净，可以迁移到 VLM 场景——在 VLM 的视觉-文本联合残差流上训练 PC 可能能检测多模态幻觉。exact NLL 避免了采样开销是一个工程优势。
- **锐评**: 把 hallucination 建模为 anomaly detection 是优雅的 framing——不需要定义什么是"正确"，只需要学什么是"正常"。PC 作为 tractable density estimator 比 VAE/diffusion 更轻量，NLL 计算是精确的不是估计的。但问题在于：PCNET 只检测文本生成中的事实性幻觉，对视觉-文本不一致的多模态幻觉完全没碰。而且 PC 的训练需要"正常"数据，如果训练数据本身含有幻觉呢？AUROC 99% 在 TriviaQA 上不算特别意外——那是 factoid QA，正常/异常的界限很清晰。在更模糊的场景下（如 long-form generation）效果可能打折扣。总的来说是好方法，但需要扩展到多模态才有更大影响。👀
- 💡 **想精读？** 运行：`读一下 论文 Hallucination as an Anomaly: Dynamic Intervention via Probabilistic Circuits`

### 13. The First Token Knows: Single-Decode Confidence for Hallucination Detection
- **作者**: 未知（从摘要未获取完整信息）
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2605.01428) | [PDF](https://arxiv.org/pdf/2605.01428)
- **来源**: 📰 HF Daily，⬆️ 未知

- **核心方法**: 提出只用第一个生成 token 的置信度就能检测幻觉，不需要多次采样或外部验证器。利用 single-decode 的 confidence score 作为幻觉信号。
- **对比方法/Baselines**: 对比了 semantic entropy、self-consistency 等需要多次采样的方法。
- **借鉴意义**: 如果 true，这是一个非常实用的发现——single decode 的 overhead 为零，可以嵌入任何生成流程做实时幻觉检测。
- **锐评**: "第一个 token 就知道"这个 claim 很 catchy 但需要仔细验证。first token confidence 能检测的更多是模型本身就不确定的 case（low confidence → possible hallucination），但模型中置信度很高但错了的情况（confident hallucination）是检测不到的。而且这个方法假定了 confidence calibration 的质量——如果模型本身 calibration 差，第一个 token 的 confidence 就没意义。需要看具体的 AUROC 数字和 false negative rate（漏检的 confident hallucinations）。💤

### 16. UniVidX: A Unified Multimodal Framework for Versatile Video Generation via Diffusion Priors
- **作者**: Houyuan Chen, Hong Li, Xianghao Kong, Tianrui Zhu, Shaocong Xu, Weiqing Xiao, Yuwei Guo, Chongjie Ye, Lvmin Zhang, Hao Zhao, Anyi Rao
- **机构**: HKUST, CUHK, Tsinghua, Beihang, Nanjing University, BAAI, Stanford
- **链接**: [arXiv](https://arxiv.org/abs/2605.00658) | [PDF](https://arxiv.org/pdf/2605.00658)
- **来源**: 🔥 HF Trending，⬆️ 80

- **核心方法**: UniVidX 用三个设计统一多模态视频生成：(1) Stochastic Condition Masking——训练时随机将模态分为 clean conditions 和 noisy targets，实现全方向条件生成而非固定映射；(2) Decoupled Gated LoRA——每种模态独立 LoRA，只在作为生成 target 时激活；(3) Cross-Modal Self-Attention——跨模态共享 K/V 但保留模态专属 Q。实例化了两个域：UniVid-Intrinsic（RGB+材质图）和 UniVid-Alpha（RGB+RGBA 图层）。
- **对比方法/Baselines**: 对比了 [[LayerDiffuse]]、[[IntrinsiX]]、[[NormalCrafter]] 等专用模型。
- **借鉴意义**: SCM 的随机模态分区策略可以迁移到其他 multimodal generation 任务——本质是一种多模态的 masked modeling。不到 1000 个视频就能训出有竞争力的结果，数据效率意外地好。
- **锐评**: 83 个点（HF Trending 80），机构阵容豪华（HKUST + Tsinghua + Stanford）。方法设计优雅——SCM + DGL + CMSA 三个组件各司其职，没有叠床架屋。用 VDM prior 做多模态生成的思路合理：视频 diffusion model 天然见过多样的视觉-运动模式，比从 scratch 训更好。但我关心的是：intrinsic decomposition 和 alpha matting 这两类任务高度相关，UniVidX 是否真的学到了物理上正确的分解，还是仅仅在 visual quality 上追平了专用模型？如果 albedo 和 irradiance 的分解在物理上不一致（比如 albedo 里残留了 shading），那说"统一框架"有点过。论文在 MAW 和 Sintel 上做了定量评估，这点不错。👀
- 💡 **想精读？** 运行：`读一下 论文 UniVidX: A Unified Multimodal Framework for Versatile Video Generation via Diffusion Priors`
